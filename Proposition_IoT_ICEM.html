Proposition d'Intégration IoT

Système d'Alerte Physique Intelligent pour Atelier

Smart Factory Alert System — Module complémentaire IoT


Projet : ICEM Quality Control System
Entreprise : ICEM Industrial Solutions — Sfax
Date : 09 Mai 2026

1. Contexte et Problématique

Le système ICEM Quality Control repose actuellement sur trois piliers technologiques :

  • Une application mobile Flutter utilisée par les techniciens pour réaliser les inspections de câbles (checklists électriques et visuelles).
  • Un module d'Intelligence Artificielle (Roboflow) intégré à l'application mobile pour la détection automatique de défauts visuels sur les câbles.
  • Un tableau de bord Web (React.js) permettant aux administrateurs et directeurs de suivre en temps réel les anomalies, rapports et tendances de production.

Le problème identifié

Dans un environnement industriel tel que l'usine ICEM, les responsables d'atelier et les superviseurs de production ne sont pas toujours devant un écran d'ordinateur. L'usine est un espace bruyant, en mouvement constant. Lorsqu'une anomalie critique est détectée par l'IA sur l'application mobile, l'information remonte immédiatement dans la base de données (Firebase Firestore) et s'affiche sur le Dashboard Web.

Cependant, si aucun administrateur ne consulte le dashboard à ce moment précis, l'anomalie peut passer inaperçue pendant plusieurs minutes, entraînant potentiellement la production de centaines de mètres de câbles défectueux avant intervention.

Il manque donc un mécanisme d'alerte physique et immédiate, perceptible dans l'atelier même sans accès à un écran.

2. Solution Proposée

Nous proposons d'ajouter un module IoT (Internet des Objets) au système existant : une station d'alerte physique placée dans l'atelier de production. Ce module est composé d'un microcontrôleur ESP32 connecté au réseau Wi-Fi de l'usine, relié à des indicateurs visuels (LEDs) et sonores (Buzzer).

Ce dispositif transforme une donnée numérique (une anomalie dans la base de données) en une alerte physique et immédiate (lumière rouge clignotante + alarme sonore), visible et audible par tous les opérateurs de l'atelier.

3. Architecture Technique

Le module IoT s'intègre à l'architecture existante du projet ICEM sans aucune modification de l'application mobile Flutter ni du Dashboard React. Il constitue un nouveau point de sortie (output) du système.

Flux de données complet

1Détection (Mobile + IA)

Le technicien scanne un câble avec l'application Flutter. Le modèle Roboflow détecte un défaut visuel. L'application crée une anomalie avec le statut "En attente" dans Firebase Firestore via le backend Node.js/Express.

2Stockage Cloud (Firebase Firestore)

L'anomalie est persistée dans la collection anomalies de Firestore avec ses métadonnées (type de défaut, sévérité, photo, technicien responsable, horodatage).

3Écoute IoT (ESP32)

Le microcontrôleur ESP32, connecté au Wi-Fi de l'usine, interroge périodiquement (toutes les 5 secondes) le backend Node.js via une route API REST dédiée : GET /api/iot/alert-status. Cette route retourne le nombre d'anomalies actives (non traitées).

4Action Physique (LED + Buzzer)

Si le nombre d'anomalies actives est supérieur à 0, l'ESP32 active la LED rouge en mode clignotant et déclenche le buzzer. Sinon, la LED verte reste allumée.

5Résolution (Dashboard Web)

L'administrateur traite l'anomalie depuis le tableau de bord React (changement de statut à "Traitée"). Lors de la prochaine interrogation, l'ESP32 détecte le changement et éteint l'alarme.

6. Valeur Ajoutée

  • Industrie 4.0 : Le projet ICEM passe d'un simple système logiciel à un véritable système Cyber-Physique, combinant IA, Cloud et IoT.
  • Réactivité Améliorée : Le temps de réaction face à une anomalie passe de plusieurs minutes (consultation manuelle du dashboard) à quelques secondes (alerte automatique dans l'atelier).
  • Zéro Impact sur l'existant : Aucune modification n'est nécessaire sur l'application mobile ou le dashboard web. Le module IoT est un ajout indépendant et non intrusif.
  • Coût Minimal : L'investissement matériel est inférieur à 50 TND, ce qui est négligeable par rapport à la valeur ajoutée en termes de traçabilité et de prévention des pertes.
  • Évolutivité : Ce prototype peut évoluer vers une vraie colonne lumineuse industrielle (Stack Light) ou un écran LCD affichant les détails de l'anomalie en temps réel.
